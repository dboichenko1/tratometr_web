# Правила формирования вебпейджа «Тратометр» (веб-версия)

Памятка для себя и для будущих сессий ИИ. Веб-версия — это самостоятельный
порт нативного Flutter-приложения `finance-tracker`. Цель: офлайн-PWA, которую
ставят на домашний экран iPhone и пользуются без интернета и без аккаунта.

## 1. Архитектура и границы
- Чистая статика: `index.html` + `app.css` + `app.js` + `xlsx.js` (+ `manifest.webmanifest`, `sw.js`, `icons/`, `tests/`).
- **Без зависимостей, без сборки, без CDN, без аналитики, без сети.** Никаких внешних `fetch`/скриптов/шрифтов/картинок. Всё инлайн/локально.
- Нет бэкенда, аккаунтов и авторизации. Данные — только локально: IndexedDB (`tratometr_db`) + зеркало в localStorage. «Разделение пользователей» = разные устройства/браузеры, а не логин.
- Все пути **относительные** — приложение должно работать и из корня домена, и из подпапки project-страницы GitHub Pages.

## 2. Service worker и кэш (САМОЕ ВАЖНОЕ)
- При **любом** изменении файлов поднимать `VERSION` в `sw.js` (`tratometr-vN` → `vN+1`). Без этого у установивших останется старый кэш (cache-first).
- Новые статические файлы добавлять в массив `SHELL` в `sw.js`.
- После деплоя приложение на устройстве обновляется при пере-открытии 1–2 раза (а не «на лету»).

## 3. Иконки приложения (iOS)
- iOS жёстко кэширует иконку домашнего экрана. При смене иконки делать **cache-bust**: добавлять `?v=N` к ссылкам `apple-touch-icon`/`icon` в `index.html` и к `src` иконок в `manifest.webmanifest`.
- Если иконка «не подтянулась» — удалить ярлык, очистить данные сайта в Safari, открыть заново и добавить.

## 4. Паритет с нативным приложением
- Поведение, форматы, тексты, отступы и цвета повторяют Flutter-приложение. Источник истины — `../finance-tracker/lib/` и его тесты `../finance-tracker/test/`.
- Деньги: `«24 855 ₽»` (узкий неразрывный пробел U+202F, копейки только у дробных), минус — U+2212. Период по умолчанию — день; тёмная тема по умолчанию; нижняя навигация по умолчанию.
- Платформенное (фоновые пуши, системная биометрия, нативный аудиоканал) на вебе работает иначе — делаем веб-аналог (локальные уведомления, PIN-замок) или опускаем; не выдаём за 1:1.
- **Намеренные расхождения с нативным (не «чинить» обратно):**
  - При импорте 1Money веб восстанавливает остатки счетов из блока «НАЗВАНИЕ/БАЛАНС/ВАЛЮТА» (выставляет стартовый остаток так, чтобы итоговый баланс счёта совпал с тем, что показывал 1Money). Нативное приложение этот блок игнорирует (импортирует только операции), поэтому там балансы = сумме операций. Веб-поведение удобнее и проверено тестами `tests/import_1money.test.js` / `tests/import_balances.test.js`. Затрагиваем только новые/пустые счета (в режиме «Заменить всё» — все из файла); счета с уже введёнными данными не трогаем.
  - У счёта есть флаг `excludeFromTotal` («Не учитывать в „Все счета“»): такой счёт исключается из общего баланса и из всех сводок по «Все счета» (списки/разбивка/графики/спидометры — всё, где `selectedAccountId==null`), но при явном выборе счёта показывается полностью; его операции не удаляются. Реализация — хелпер `accExcluded()`; покрыто `tests/exclude_account.test.js`. В нативном приложении флага нет.

## 5. Тесты (обязательно перед коммитом)
- Логику держим в чистых функциях, чтобы её можно было тестировать без DOM: `evaluateSpendingAlert`, `evaluateLimitAlert`, `parse1MoneyCsv`, `resolveOnAccent`, `setAccentColor`, форматтеры, `rangeFor`, `nextOccurrence`, балансы.
- Тесты — в `tests/` на Node (DOM подменяется шимом `_harness.js`). Запуск: `node tests/run.js`. Всё должно проходить.
- Новые тест-кейсы при возможности портируем из нативных тестов (точные ожидаемые значения).

## 6. Безопасность (репозиторий публичный)
- Никаких PII/секретов/рабочей почты в коде, коммитах и иконках.
- Единственный sink `innerHTML` — это SVG-разметка из чисел и цветов, цвета прогоняются через `safeColor()`. Пользовательский текст (названия, комментарии) выводить только текстовыми узлами (`createTextNode`), не через `innerHTML`.
- Никаких `eval`/`new Function`/`document.write`.

## 7. Совместимость данных
- Не ломать схему `S.data`. `normalizeLoaded()` домердживает недостающие настройки дефолтами — старые сохранённые копии должны открываться.
- Сид (`seedData`) менять осторожно (влияет только на первый запуск).

## 8. Код-стиль
- Вандл-JS, без фреймворков. DOM строим хелпером `h(tag, props, ...children)`.
- Тема — через CSS-переменные, проставляет `applyTheme()`. Иконки интерфейса — инлайновые SVG (`ICONS`/`icon()`); иконки категорий — эмодзи.

## 9. Чек-лист перед коммитом
1. `node --check app.js && node --check sw.js`
2. `node tests/run.js` — зелёно
3. грэп безопасности (PII / `eval` / внешние `https?://` / `fetch`)
4. поднят `VERSION` в `sw.js`, новые файлы добавлены в `SHELL`
5. коммит в `main` → push (GitHub Pages раздаёт `main` / `root`)
